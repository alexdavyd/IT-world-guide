#  Общие подходы к масштабированию

Масштабирование — это способность системы увеличивать производительность при росте нагрузки. Выделяют два основных направления:

- **Вертикальное масштабирование (Scale Up)** — увеличение мощности существующего сервера (больше CPU, RAM, дисков). Просто, но есть физический предел и высокая стоимость.
- **Горизонтальное масштабирование (Scale Out)** — добавление новых узлов в кластер. Более гибко, требует распределения данных и управления согласованностью.

Основные техники горизонтального масштабирования баз данных:
- **Репликация** — копирование данных на несколько серверов для повышения отказоустойчивости и распределения чтения.
- **Шардирование** — разделение данных на части (шарды), каждая хранится на отдельном сервере.
- **Партиционирование** — логическое деление таблиц внутри одной БД (часто используется вместе с шардированием).

# Сравнение

| Характеристика | Репликация | Шардирование |
|----------------|------------|--------------|
| Данные на каждом узле | Полная копия | Часть данных |
| Цель | Отказоуст., распределение чтения | Распределение записи и объёма |
| Сложность | Низкая | Высокая |
| Масштабирование записи | Нет (одна точка записи) | Да |
| Масштабирование чтения | Да | Да (на каждом шарде своя репликация) |

В реальных больших системах эти методы комбинируют: данные шардируют, а внутри каждого шарда используют репликацию для надёжности и балансировки чтения.
